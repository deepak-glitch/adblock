# Screenshots

Real device captures for the project READMEs live here. The READMEs currently
use SVG **mockups** (`docs/banner.svg`, `docs/dashboard.svg`) so nothing is ever
a broken image — but real photos look better. Adding them is a 30-second drag-and-drop.

## How to add yours (no git needed)

1. Go to this folder on GitHub: **`deepak-glitch/adblock` → `docs/img`**
2. **Add file → Upload files**, drag your screenshots in, and **name them exactly** as below.
3. **Commit** straight to `main`.

That's it — then ping me (or edit the READMEs) to swap the `.svg` references for these.

## Filenames the READMEs expect

| File | What it should show |
|------|---------------------|
| `dashboard.jpg` | App main screen — status **Active** + the four stat cards |
| `install-downloader.jpg` | The Downloader app with `tinyurl.com/23hujy5w` typed in |
| `max-playing.jpg` | Max playing an episode with ads blocked |
| `about-fireos.jpg` | `Settings → My Fire TV → About` showing **Fire OS** |

(Photos of the TV screen are fine. JPG or PNG both work.)

## Swapping a mockup for a real shot

In `README.md` / `firetv/README.md`, change:

```html
<img src="docs/dashboard.svg" ...>
```

to:

```html
<img src="docs/img/dashboard.jpg" ...>
```

Tell me once they're uploaded and I'll wire them into both READMEs (and into the
install steps) for you.
